# Virtual Classroom Companion System (虛擬教室夥伴系統)

> 專案計劃書 - 2026-02-13

## 專案概念

將真實教室映射到虛擬空間，每位同學擁有專屬虛擬夥伴（類似 Animal Crossing），透過 NFC 簽到認養，結合疲勞偵測實現互動，鼓勵出席與專注。

## 六大模組

### 1. Classroom 3D Scan（教室 3D 掃描）

**目標**：掃描真實教室，建立虛擬空間基底

**硬體**：
| 項目 | 規格 | 數量 | 預估成本 |
|------|------|------|----------|
| OAK-D 相機 | 已有 (Horizon/Gravity) | 2 | $0 |
| 三腳架/雲台 | 可旋轉掃描 | 1 | ~$50 |
| (可選) OAK-D Pro | 更好的深度精度 | 1 | ~$300 |

**軟體**：
| 軟體 | 用途 | 授權 |
|------|------|------|
| DepthAI 3.x | 深度圖擷取 | 免費 |
| Open3D | 點雲處理/拼接 | 免費 |
| CloudCompare | 點雲編輯/清理 | 免費 |
| Meshroom / COLMAP | 多視角重建 (可選) | 免費 |

**輸出**：
- 點雲檔案 (.ply, .pcd)
- 3D 網格模型 (.obj, .fbx)

---

### 2. Virtual Space Mirror（虛擬空間鏡像）

**目標**：在遊戲引擎中重建教室，實時反映真實狀態

**硬體**：
| 項目 | 規格 | 數量 | 預估成本 |
|------|------|------|----------|
| 開發電腦 | 需 GPU (已有 Mac Mini/PC) | 1 | $0 |
| 伺服器 (可選) | 運行 Unity 後端 | 1 | 雲端費用 |

**軟體**：
| 軟體 | 用途 | 授權 |
|------|------|------|
| Unity 3D | 虛擬教室渲染引擎 | 免費 (個人版) |
| Blender | 3D 模型編輯/優化 | 免費 |
| WebGL Build | 網頁版虛擬教室 | Unity 內建 |
| Photon/Mirror | 多人連線 (可選) | 免費/付費 |

**功能**：
- 匯入掃描的教室模型
- 即時顯示座位狀態（有人/無人）
- 虛擬夥伴活動空間

---

### 3. Robot Classmate（虛擬夥伴）

**目標**：每位同學擁有專屬虛擬寵物/機器人夥伴

**軟體**：
| 軟體 | 用途 | 授權 |
|------|------|------|
| Unity | 角色動畫/AI | 免費 |
| Ready Player Me | 虛擬角色生成 (可選) | 免費 API |
| Mixamo | 角色動畫素材 | 免費 |

**遊戲機制** (借鏡 Animal Crossing)：
| 機制 | 說明 |
|------|------|
| 能量值 | 出席 +10，缺席 -20/天 |
| 心情 | 根據互動頻率變化 |
| 等級 | 累積出席解鎖外觀 |
| 互動 | 可餵食、對話、玩遊戲 |
| 禮物 | 連續出席獲得獎勵 |

**資料庫**：
| 軟體 | 用途 |
|------|------|
| Supabase / Firebase | 學生資料、夥伴狀態 |
| PostgreSQL | 自建資料庫 (可選) |

**目標**：偵測學生疲勞狀態，虛擬夥伴即時反應

**硬體**：
| 項目 | 規格 | 數量 | 預估成本 |
|------|------|------|----------|
| OAK-D 相機 | 已有 | 2-3 | $0 |
| Raspberry Pi 5 | 已有 | 2-3 | $0 |

**軟體**：
| 軟體 | 用途 | 狀態 |
|------|------|------|
| fatigue_detector.py | 疲勞偵測 | ✅ 已完成 |
| gaze_detector.py | 視線追蹤 | ✅ 已完成 |
| WebSocket Server | 即時推送狀態 | 需開發 |

**互動設計**：
| 偵測事件 | 虛擬夥伴反應 |
|----------|--------------|
| 眼睛閉合 | 夥伴揮手提醒 |
| 頭部低垂 | 夥伴跳到你面前 |
| 長時間專注 | 夥伴開心跳舞 |
| 離開座位 | 夥伴變傷心 |

### 5. Virtual Projection（虛擬投影）

**目標**：將虛擬教室投影到真實空間

**硬體選項**：

| 方案 | 硬體 | 預估成本 | 效果 |
|------|------|----------|------|
| A. 大螢幕 | 電視/投影機 | $200-500 | 2D 展示 |
| B. AR 眼鏡 | Meta Quest 3 / HoloLens | $500-3500 | 沉浸式 AR |
| C. 手機 AR | iOS/Android + ARKit/ARCore | $0 | 隨身查看 |
| D. 桌面投影 | Looking Glass Display | $300-2000 | 裸眼 3D |

**推薦方案**：
- **初期**：大螢幕 + 手機 AR App
- **進階**：Meta Quest 3 MR 混合實境

**軟體**：
| 軟體 | 用途 |
|------|------|
| Unity AR Foundation | 跨平台 AR |
| Vuforia | AR 追蹤 (可選) |
| Meta SDK | Quest 開發 |

---

### 6. NFC Check-in（NFC 簽到認養）

**目標**：學生用 NFC 卡簽到，綁定虛擬夥伴

**硬體**：
| 項目 | 規格 | 數量 | 預估成本 |
|------|------|------|----------|
| NFC 讀卡機 | PN532 / RC522 | 1-2 | ~$10-20 |
| NFC 卡片/貼紙 | NTAG215 | 30+ | ~$0.5/張 |
| Raspberry Pi | 已有 | 1 | $0 |
| (可選) 手機 NFC | 學生自己的手機 | 0 | $0 |

**軟體**：
| 軟體 | 用途 |
|------|------|
| nfcpy (Python) | Pi 上讀取 NFC |
| Web NFC API | 手機瀏覽器讀取 |
| Flask/FastAPI | 簽到後端 API |

**簽到流程**：
```
學生刷 NFC → 讀取 UID → 查詢資料庫 →
    ├─ 新用戶 → 創建夥伴 → 選擇外觀
    └─ 舊用戶 → 更新出席 → 夥伴能量 +10
```
